## 0. 交易口径（已确认）

**方向**：多空都做，采用**分离式两套系统**（多头引擎 + 空头引擎，见第 1 节）。

**品种**（2026-07-03 确认）：**中证500 + 中证1000**，回测三口径并行：500 单独、1000 单独、50/50 等权拟合指数。执行合约 IC/IM。

品种确定后，**贴水 carry 的不对称性**论据更加成立——IC/IM 恰是贴水最深的两个品种（长期年化贴水中枢约 5–10%，分年份波动大）：

- 做多深贴水合约自带正 carry（贴水收敛收益），做空要先跨过同等幅度的 carry 逆风；
- 含义：**同一个信号，做多 IC/IM 是顺风局，做空是逆风局**。空头信号必须显著强于多头信号才值得开仓——这是"两套系统"最硬核的论据；
- 回测规则：空头段计入贴水 carry 成本，多头段计入 carry 收益（用当期实际基差，不用长期均值）。

## 1. 多空分离双引擎架构（方向一的结构层）

### 1.1 为什么 A 股适合多空两套系号

1. **动力学不对称**：上涨多为流动性/散户情绪驱动（慢热、扩散），下跌多为去杠杆/流动性收缩（急促、高波动）。捕捉二者的特征集不同：涨的前兆是"性价比+拐点确认"（慢变量），跌的前兆是"拥挤+脆弱性+触发"（快变量）；
2. **成本不对称**：贴水 carry（见第 0 节）+ 做空的尾部风险被政策托底截断（政策底/国家队），空头开仓门槛理应更高、持有期更短、止损更紧；
3. **现有系统已隐含这个思想**：hybrid20 本来就是不对称的——多头完全相信主信号，空头需金融信号不反对；状态机阈值也不对称（开多 0.35 vs 开空 −0.15）。用户的直觉是把这个雏形显式化、彻底化；
4. **可分别评价**：空头信号的价值往往不在独立盈利而在避险。混在一个对称系统里评价，空头贡献会被多头段淹没。

### 1.2 三个候选架构

| 架构 | 描述 | 评价 |
|---|---|---|
| (a) 对称单系统 | 一个连续因子，对称阈值双向（citic40d 式） | 简单，但把两种不同动力学硬塞进一个模型 |
| (b) 不对称单系统 | 一个因子，非对称阈值 + 空头确认（hybrid20 现状） | 中间态；空头仍依赖与多头同源的信号 |
| (c) 分离双引擎 ⭐ | 多空各自独立的信号集、频率、阈值、评价，仅在执行层合成 | 结构上匹配动力学不对称；代价是空头引擎样本少、过拟合风险需管理 |

**推荐 (c)，演进式落地**：v1 多头引擎 = 现有信号多头段，空头引擎 = 现有信号空头段 + 环境门槛——先把框架立起来（评价、成本、合成规则），再逐步给两个引擎喂各自的专属信号（第 2 节清单已按引擎归属标注）。这样每一步都可回测验证，不是推倒重来。

### 1.3 双引擎设计要点

```
共享因子池（第 2 节） ──→ 多头引擎：慢频（周~月）、趋势确认型、门槛较低
                     │      信号族：风格轴多头、基差修复、广度扩张、ETF 逆势申购、ERP 高分位
                     │      输出 long ∈ {0, +1}（可扩展仓位维度）
                     │
                     └──→ 空头引擎：快频（日~周）、"干柴+火花"两段式、门槛高
                            环境层（干柴）：两融增速极值、题材集中度、微盘拥挤、风格 z 极端正、成交额分位极高
                            触发层（火花）：炸板率飙升、涨停溢价转负、广度背离、基差急剧走深、龙头补跌
                            保护层：深贴水 carry 门槛、超跌位置禁空
                            输出 short ∈ {0, −1}

合成（执行层）：net = long + short ∈ {−1, 0, +1}；同时触发 → 0（亢奋末期常见"趋势未完+脆弱性已现"，宁可空仓）
```

**评价分开**：
- 多头引擎 vs 满仓基准：超额收益 + 回撤改善 + 空仓段躲过的下跌；
- 空头引擎双口径：独立盈亏（含 carry 逆风）+ **避险价值**（标的指数下跌 >5% 的月份命中率、加入后组合 MaxDD 改善幅度）。空头引擎即使独立盈亏平平，只要避险价值显著就值得保留。

### 2.2 六对配对冗余度实证与精简建议（2026-07-03 快速分析）

用户质疑"为什么要这 12 条指数"。历史答案：2026-06-18 原设计就是"导出 CSV 里相邻列全部配对、等权"，**没有更深的选择依据**。实证答案（用现有输出 `equal_weight_signal_20d40z.csv` 六对信号 2020-03~2026-06 + PG 内 000905/000852 收盘价；全样本、重叠窗口粗口径，IC 绝对值偏乐观，**排序结论可用**）：

1. **四个宽基对高度冗余**：300/500/1000/2000 对两两相关 0.72–0.88（日变化口径 0.55–0.79）。六对 PCA：PC1 解释 62%，**六对 ≈ 1.6 个独立信号**——本质是同一根成长/价值轴在四个市值段的拷贝；
2. **创业板对、科创对是仅有的分散来源**（与宽基对相关 0.2–0.4），但预测力弱：对 500/1000 未来 20 日收益的 Spearman IC，创业板对 ≈ 0.13，**科创对 0.02–0.07 接近噪声**（历史短、成分窄）；
3. **精简子集反而更强**：对 50/50 拟合标的，1000 对单独 IC 0.35 > "500+1000 两对" 0.32 > 六对等权 0.30 > 创业板对 0.13 > 科创对 0.03。信号与标的的市值段匹配有明确收益；
4. **三条信号线中度同质**：equal_weight / citic40d / hybrid20 连续因子两两相关 0.60–0.69——有互补成分（citic40d 的周期/消费、金融因子贡献差异）但同源。

**精简建议**（信号设计层）：

- **核心对**：500、1000、2000 三对（与标的市值段匹配；2000 对保留是因为它对小票情绪的延伸敏感度最高且 IC 第二）；
- **观察对**：300 对不入等权，改造成**大小盘风格分歧变量**（300 对与 1000 对的差值 = 大盘成长/价值与小盘成长/价值的分歧度，归入环境变量池）；创业板对低权重候选（低相关弱信号，集成时可小权重）；
- **剔除**：科创对；
- 相应地，**Wind 代码最小核对集 = 8 条**（500/1000/2000 三对 6 条 + 300 对 2 条），可选 +2（创业板对），科创对 2 条不再需要。

**正式结论待 Phase 2**：以上是讨论级证据；配对取舍最终由 walk-forward OOS 框架裁决（把"六对等权 vs 三核心对 vs 单 1000 对"作为 Phase 2 参数重扫的对照组之一）。

### 2.3 指数对成分透视综合发现（2026-07-02 用户 15 对穿透 + 2026-07-03 分对透视）

用户对 15 对风格指数做了等权穿透（`/home/elfbob/exchange/20260702/指数成分/`），并与中信五风格对比；本节补充了推荐四对（300/500/1000/2000，中证系）的分对行业透视。合并发现：

1. **风格价差的很大部分是行业对赌**。各对"成长腿−价值腿"的行业主动暴露（单边）：300对 **73%**、500对 64%、1000对 52%、2000对 45%。多头行业清一色电子+通信/计算机/新能源，空头行业大市值段是银行/非银，小市值段变成医药/公用——**"价值"在不同市值段含义都不同**；
2. **对间成分零重叠但信号高相关**。四对成长腿两两 Jaccard = 0（中证风格系列在互斥母指数上选样），而信号相关 0.72–0.88——高相关**不是成分重叠造成，而是真正的市场级风格共振**。这支持风格轴信号的有效性，同时说明"多对等权"的分散天然有限（驱动因子同一个）；
3. **编制口径各说各话**：茅台/紫金矿业在穿透成长 Top10，比亚迪/京东方在价值侧；**CI稳定与"价值"余弦相似度仅 0.09**（CI稳定实为长江电力 12.6% 领衔的公用红利篮子），CI金融才与价值高重合（cosine 0.69）。含义：hybrid20/citic40d 的"成长/稳定"轴 ≈ 科技成长 vs 防御红利；equal_weight 的"成长/价值"轴 ≈ 科技 vs 银行——**三条线画的其实是两根不同的轴**，这解释了三线相关只有 0.6–0.69；
4. 300/500 对成长∩价值有 8% 权重交集（旧编制方法允许一票两属），1000P/2000P（932 系新编制）互斥。

**对信号设计的含义**：指数对信号 = 纯风格轮动 + 行业轮动 + 编制怪癖 的混合体。它依然是有效的情绪代理（IC 实证在前），但"简化"的代价已量化——这引出 2.4 的自建方案。

### 2.4 自建风格篮子：从指数对到直接选股（用户提出，方向一的深化）

**动机**：指数对是别人定义的简化。自建 = 用选股因子直接从市场构造成长/价值组，把口径、universe、正交性抓在自己手里。最大的增量价值是**把"风格轴"与"行业轴"分离成两个独立信号**——指数对做不到（见 2.3 发现 1）。

**数据可行性（2026-07-03 实查，全部满足）**：
- `stock_financial` 带 **ann_date + available_at** → point-in-time 纪律有基础；4.79M 行、2003 起、95.8% 覆盖 ≥9 季度；
- `stock_share_capital` 62,922 行（effective_date + available_date）→ 历史总股本可算市值、BP/EP（**绕开 stock_indicator 只有 2025 年起的短板**）；
- `industry_classification` CITIC 67 期月度快照（2021-01 起）→ 行业中性 2021 后严谨，之前用最早快照近似；
- 个股日线 35 年含退市票（6,442 ticker）→ survivorship 可控（需抽查验证）。

**方案设计**：

**universe 原则（2026-07-03 用户定调）**：信号是对市场的刻画，标的（500/1000）只是回测载体——**是否把信号生成限定在标的市值段，是待检验假设，不预先定论**。管线设计：打分全市场计算，universe 仅为末端过滤器，多套并行输出零边际成本，Phase 2 回测框架统一裁决。候选 universe 集（嵌套设计，分别回答"前 300 要不要""下探多深"）：

- U0 全市场（轻过滤：剔 ST、上市 <6 个月；过滤条件本身保持极简，避免把决定藏进过滤器）
- U1 剔除前 300（全中小盘）
- U2 301–1800（标的段；市值排名用总股本×价格近似，误差与校准见下）
- U3 301–3800（标的段 + 2000 段）

**中信风格的机制（第五套，与前四套本质不同）**：CI 风格指数**不打个股因子分、不选样**，而是以中信**三级行业**为单位给每个行业贴风格标签，风格指数 = 标签下全部股票。标签划分的要点：

- **CI成长** = 新兴产业行业集合（军工、光伏/风电/核电、生物医药、计算机、通信、电子、互联网、环保/水务/燃气等 39 个三级行业）——**不含任何财务成长性判断**；
- **CI稳定** = 纯公用+基建运输（火电/水电/电网、建筑施工、公路/铁路/港口/机场、电信运营，仅 11 个行业）——证实透视发现：**"稳定"是防御公用红利，与"价值"是两个概念**（cosine 0.09 的根源）；
- **CI金融** 含地产（银行/证券/保险/信托 + 住宅地产/园区/房地产服务）；

- 成长分：SalG = 12 季度滚动营收 TTM 对时间回归斜率÷|均值|（`income.gr_ttm`）；ProG = 同法净利 TTM（`profit_ttm`；**扣非科目 Wind 侧暂缺，v1 不扣非并标注**，升级查 CSMAR F 码或回填）；ΔROE = ROE_TTM 同比变化（`fa_roe_ttmavg`；原版用单季，TTM 同比为平滑近似）；

- 处理：5%/95% 缩尾 → 截面 Z → /√n 合成；缺失值行业均值填充（借 Gen1 明确规则）；PIT 用 `ann_date`/`available_date`；
- 分桶：硬分桶（Top/Bottom 30% 等权）为主，**Gen3 式概率软分桶**（sigmoid 概率加权）为对照变体，进 Phase 2 扫描；
- **最大工程点**：财务序列是 CSMAR（F 字段码，2003–2025Q1）+ Wind（命名字段，2025Q2 起）两源拼接，需一层**科目映射**并做拼接期连续性校验。

## 3. 方向二：参数优化方法论

### 3.1 现有评价的四个问题（`signals/hybrid20/optimize_signal.py`）

1. **年化被夸大**：`evaluate_signal` 剔除 signal==0 日后仍 `mean() × 245`，空仓日被从分母去掉。改为全日历持仓序列（空仓日收益 = 0）；
2. **零成本假设**：无手续费/滑点/carry。状态机月均切换次数 × 成本决定短窗参数存活与否；空头段的贴水 carry 是数量级更大的成本项；
3. **全样本内优化**：网格搜索与 IC 加权均在全样本上做；
4. **单一标的评价**：只用沪深300，与实际用途（IM 为主）不对齐。

### 3.2 评价框架（先修秤，再称重）

- **Step 0 修口径**：全日历收益序列；年化/Sharpe/MaxDD/Calmar；多头段与空头段**分开报告**（对齐双引擎架构）；成本 = 手续费+滑点（期货单边约 3bp）+ 空头段贴水 carry；成交假设 T 收盘信号 → T+1 收盘成交（开盘价作敏感性对照）；
- **Step 1 显著性基准**：满仓持有、只多版、同换手随机信号 bootstrap（1000 次）——指标须落随机分布右尾（p<0.05）；
- **Step 2 样本切分**：walk-forward 滚动 3 年训练/1 年测试拼接 OOS 曲线；或 2014–2020 训练 / 2021–2023 验证 / 2024–2026 留出。**对外数字只认 OOS**；
- **Step 3 扫描与稳健性**：lookback ∈ {5,10,20,40,60} × z_window ∈ {2×,3×,5×lookback, 250} × 平滑 ∈ {0,3,5,10}；阈值先对称约束降自由度，不对称只有 OOS 稳定复现才保留；**选 Sharpe 高原不选尖峰**（输出热图）；
- **Step 4 结构性修复**：`build_basket` 的 `iloc[0]` 归一改为等权日收益合成（当前实现非 point-in-time，追加数据会改写历史信号）；IC 加权改滚动窗口；250d z 窗口首年无信号，各变体评估期对齐后再比。

### 3.3 参数的逻辑先验

- z 窗口的含义是"相对多长的历史算异常"：250d=年度异常（趋势跟随特性），40d=两月异常（快速均值回归特性）——**是两种策略假设，不是同一策略的两个参数点**。先定引擎目标频率（多头引擎慢、空头引擎快），再定各自窗口量级；
- `z_window = 2×lookback` 约定无先验依据，作为参数线检验而非默认；
- tanh(z) vs tanh(z/2) 仅压缩斜率差异，统一约定即可，不值得优化；
- 情绪类指标注意**分位区间依赖**：中性区间往往趋势跟随有效，极值区间往往均值回归有效——同一指标两种用法要在设计时显式区分，避免网格扫出"平均正确、处处平庸"的参数。
